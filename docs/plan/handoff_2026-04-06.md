## Key Facts & Decisions

| Item | Value / Decision |
|------|-----------------|
| `clk_use_local` default | Absent from localStorage = `true` (local time). Never writes `"true"` unless user explicitly selects local after having switched to manual. |
| Local time implementation | `new Date().getHours()` / `.getMinutes()` — not UTC + offset. Handles DST automatically. |
| Gemini MCP env var | `GOOGLE_API_KEY` sourced from `~/Claudette/Cowork/projects/.env` via `~/.zshrc`. Requires shell restart or `source ~/.zshrc` to activate. |
| `--opacity-dim` | `0.8` — resting opacity for icon/UI elements with hover-to-full pattern. Added to `colors.css` OPACITY section. |
| Gemini audit pattern | Pattern 1 (Architecture Audit) from `GEMINI-CONSULTANCY.md`. Flash model, temperature 0.3. |
| One adjudicated false positive | `ls()` vs `ls.get()` difference between `clock.js` and `controls.js` — intentional IIFE-scoped helpers, not drift. |

## Known Constraints

- `clk_use_local` absent from localStorage must always be treated as `true`. If future code adds a new "reset" or "clear all" localStorage function, it must NOT write `"false"` as a default — absence is the correct default state.
- Gemini MCP requires `GOOGLE_API_KEY` in shell environment. If a new terminal session doesn't have it, `source ~/.zshrc` first.
- The controls icon `filter: brightness(0) invert(1)` makes it pure white — only works against a dark background. If the page ever gains a light mode background where the links panel appears, this will need revisiting.
